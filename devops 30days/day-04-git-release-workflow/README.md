# Day 04 - Git And Release Workflow

## Target

Understand how code safely becomes production.

## Learn Deeply

- Branching and pull requests.
- Commit quality and release notes.
- Tags and immutable release references.
- Revert vs reset.
- GitOps basics: Git as source of truth.
- Rollback thinking.

## Hands-On Lab

1. Create a branch.
2. Make a small documentation change.
3. Run `git diff`.
4. Write a release note with risk and rollback.

## Interview Angle

Say:

```text
I treat every production change as a controlled release: review, CI checks, image tag, deploy, verify, rollback plan.
```

## AWS/GCP Mapping

Git flow is cloud-independent. Cloud Build, GitHub Actions, CodeBuild, and Jenkins all depend on clean Git release discipline.

## Daily Motivation

Interviewers trust engineers who understand change control.
