- Variable.

1. Identify provider.
2. Identify variables.
3. Identify resources.
4. Identify outputs.
5. Explain where state should live.

## Interview Angle

Say:

```text
I treat terraform plan as a blast-radius report. I never apply unexpected destroy changes blindly.
```

## AWS/GCP Mapping

Terraform workflow is the same for AWS and GCP. Providers and resources differ; state discipline is identical.

## Daily Motivation

Terraform skill is safe change management, not just writing `.tf` files.
