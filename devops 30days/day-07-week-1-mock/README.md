# Day 07 - Week 1 Mock And Revision

## Target

Connect Linux, Bash, Python, Git, Docker, and Compose into one story.

## Deep Revision

You should now be able to explain:

- How to inspect a slow Linux server.
- How to write safe Bash scripts.
- When Python is better than Bash.
- How a Git change reaches production.
- How Docker build/runtime issues differ.
- How containers communicate in Compose.

## Hands-On Lab

Build a mini diagnostic tool:

- List running Docker containers.
- Show unhealthy/exited containers.
- Print last 20 logs for failed containers.
- Exit non-zero if failures exist.

## Interview Angle

Prepare this story:

```text
I automated health inspection by combining Linux commands, shell/Python scripting, and Docker logs. This reduced manual debugging and made failures easier to identify.
```

## AWS/GCP Mapping

This foundation applies before cloud-specific debugging. EC2, GCE, ECS, GKE, and EKS still need OS/container inspection.

## Daily Motivation

Week 1 is your foundation. Strong basics beat shallow tool memorization.
