# Day 23 - Multi-Cloud Mapping

## Target

Use your GCP knowledge to speak AWS confidently.

## Learn Deeply

- Project vs Account.
- Service Account vs IAM Role.
- VPC differences.
- GKE vs EKS.
- Cloud SQL vs RDS.
- Artifact Registry vs ECR.
- Cloud Logging vs CloudWatch.
- Cloud Audit Logs vs CloudTrail.

## Hands-On Lab

Convert this GCP architecture to AWS:

```text
GKE + Artifact Registry + Cloud SQL + Cloud NAT + Cloud Load Balancer + Cloud Monitoring
```

AWS version:

```text
EKS/ECS + ECR + RDS + NAT Gateway + ALB + CloudWatch
```

## Interview Angle

Say:

```text
My GCP knowledge transfers because the layers are the same: identity, network, compute, storage, deployment, observability, and security.
```

## AWS/GCP Mapping

Focus extra attention on AWS IAM and VPC because those differ most from GCP.

## Daily Motivation

You are not starting AWS from zero. You are translating cloud concepts.
