# Day 29 - Full Mock Interview

## Target

Simulate a real DevOps/Cloud Engineer interview.

## Format

45 to 60 minutes:

- 5 Linux/Python questions.
- 5 Docker/Kubernetes questions.
- 5 CI/CD/Terraform questions.
- 5 AWS/GCP questions.
- 5 production scenario questions.
- 1 project deep dive.

## Rules

- Answer out loud.
- Use VERDICT for scenarios.
- Keep first answer under 90 seconds.
- Add details only when interviewer asks.
- Record yourself if possible.

## Hands-On Lab

Pick 5 weak questions and rewrite better answers.

## Interview Angle

Say:

```text
If I do not know the exact service-specific answer, I still debug by identity, network, resources, dependencies, infra health, connectivity, and telemetry.
```

## AWS/GCP Mapping

Be honest: GCP is stronger, AWS is mapped and actively practiced. Then prove with clear AWS equivalents.

## Daily Motivation

Mock interviews expose weak areas before real interviews do.
