# Round 2 - Technical Questions

## Focus Areas

- Linux
- Scripting
- Docker
- Kubernetes
- Terraform
- CI/CD
- GCP/AWS mapping

## Key Questions

### 1. Debug slow Linux server.

Answer with CPU, memory, disk, process, logs, network, recent change.

### 2. Bash vs Python.

Bash for simple command orchestration. Python for APIs, JSON, retries, testing, maintainability.

### 3. Docker build fails.

Read exact failing layer, check base image, dependency, network, build context, disk.

### 4. Container exits.

Check logs, exit code, command, env vars, permissions, dependencies.

### 5. CrashLoopBackOff.

Use logs previous, describe, events, env/secrets, resources, dependency.

### 6. Kubernetes service not reachable.

Check selector, endpoints, targetPort, readiness, DNS, network policy.

### 7. Terraform state.

State maps config to real resources. Use remote backend and locking.

### 8. Terraform drift.

Detect with plan. Fix by config update, import, or reverting manual changes.

### 9. CI/CD pipeline.

Lint/test/build/scan/push/deploy/smoke test/rollback.

### 10. GCP project to AWS.

GKE->EKS, Artifact Registry->ECR, Cloud Build->CodeBuild, Cloud SQL->RDS, LB->ALB, Monitoring->CloudWatch.
